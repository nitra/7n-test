## Огляд

Цей файл налаштовує середовище для тестування мутацій з використанням Vitest. Він оптимізує роботу Stryker, забезпечуючи швидкість за рахунок фокусування на релевантних тестових сценаріях, та зберігає результати згідно з конфігураціями `mutation.json` та `incremental.json`.

## Поведінка

1. Визначає, що засіб для тестування, який буде використовуватись для виявлення мутацій, — це Vitest, з посиланням на конфігураційний файл `vitest.config.mjs`.
2. Встановлює аналіз покриття таким чином, що Stryker запускає лише ті тести, які стосуються мутованої лінії коду, що сприяє збільшенню швидкості порівняно з аналізом усього тестового набору.
3. Встановлює тимчасову директорію для звітів Stryker у `reports/stryker/.tmp`.
4. Конфігурує репортери для виводу результатів у форматах JSON та clear-text, зберігаючи детальні дані мутацій у `reports/stryker/mutation.json`.
5. Вмикає інкрементальний режим, що дозволяє відновлювати результати між запусками, що підвищує швидкість при повторних прогонах.
6. Посилається на файли `mutation.json` та `incremental.json` як на конфігурації, які використовуються цим кодом.
