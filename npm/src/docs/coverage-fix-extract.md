## Огляд

Реалізує витягування даних про вцілілих мутантів із великого файлу `COVERAGE.md`. Цей компонент адаптує результати покриття коду, надаючи LLM-оркестратору невеликі, цільові порції інформації: або загальний індекс (`index`) для визначення фан-ауту, або контекст (`slice --file <path>`) для аналізу окремого файлу. Операція є виключно **read-only** — вона лише парсить, не виконує жодних змін у файловій системі. Компонент забезпечує високу стійкість, перехоплюючи можливі помилки (fail-safe) та не використовує кешування.

## Поведінка

Поведінка:
parseSurvivedBlock витягує JSON-масив вцілілих мутантів з тексту `COVERAGE.md` на основі пошуку секції `## Вцілілі мутанти` та відповідного JSON-блоку.
readSurvived читає вміст файлу `COVERAGE.md` із кореня проєкту та повертає структуровані групи вцілілих.
buildIndex згортає структуровані групи вцілілих у компактний масив, що містить ім'я файлу та кількість вцілілих мутантів.
runCoverageFixCli керує командним інтерфейсом, викликаючи відповідні функції для виведення компактного індексу або генерації промпта для одного файлу.

## Публічний API

- parseSurvivedBlock — Зчитує секцію `## Вцілілі мутанти` з `COVERAGE.md`, вибирає вміст першого блоку формату JSON та перетворює його на масив об'єктів.
- readSurvived — Отримує з кореня проєкту вміст `COVERAGE.md` та структурує дані про вцілілих мутантів.
- buildIndex — Агрегує структуровані групи вцілілих мутантів у компактний масив, де кожен елемент містить ім'я файлу та список мутантів, що в ньому вижили.
- runCoverageFixCli — Інструмент командного рядка, який може вивести компактний JSON-масив індексу або, за вказаним шляхом, надати промпт для аналізу даних одного файлу, використовуючи лише `COVERAGE.md`.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
