## Огляд

При виклику команди `n coverage --fix`, система запускає pi-агента для автоматичного написання тестів, які покривають вцілілі мутанти Stryker. Агент аналізує контекст кожного мутанта, використовуючи спеціалізовані моделі (`CLOUD_MAX` або `N_CURSOR_COVERAGE_FIX_MODEL`), для створення відповідних тест-файлів. Процес ініціюється через публічні функції `fixSurvivedMutants` та `buildFixPrompt`. Дана робота є read-only; вона перехоплює потенційні помилки, використовуючи fail-safe механізми, і не генерує кешованих даних.

## Поведінка

Поведінка
fixSurvivedMutants запускає pi-агента для генерації тестів, які покривають вцілілі мутанти Stryker, використовуючи наданий список мутантів та кореневий шлях проєкту.
buildFixPrompt формує детальний промпт для агента, описуючи кожен вцілілий мутант разом із контекстом вихідного файлу.

## Публічний API

fixSurvivedMutants — запускає pi-агента для генерації юніт-тестів на основі вцілілих мутацій.
buildFixPrompt — створює деталізований запит для агента, який містить список вцілілих мутантів, згрупованих за файлами, разом з контекстом з вихідного коду навколо кожної мутації.

## Гарантії поведінки

- Read-only: не виконує операцій запису (ФС/БД).
- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
