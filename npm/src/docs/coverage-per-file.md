## Огляд

Файл виконує повне сканування тестів Vitest у єдиному проході, генеруючи звіт про покриття для кожного файлу та провали тестів. Він базується на конфігураціях, визначених у `package.json` та `test-results.json`. Логіка дозволяє знаходити (`findSourceFiles`), вимірювати (`measureCoveragePerFile`) покриття та отримувати список неповних файлів (`getUncoveredFiles`), при цьому свідомо пропускаючи каталоги `.git` та `node_modules`, а також гарантуючи стійкість до помилок.

## Поведінка

Поведінка:
measureCoveragePerFile запускає Vitest у єдиному проході, генеруючи звіт про покриття за кожним файлом та дані про провали тестів. При цьому ігнорує файли в .git та node_modules, а також тимчасово створює та видаляє каталог для результатів.
getUncoveredFiles фільтрує наданий список файлів, вибираючи ті, які мають покриття нижче заданого порогу.
findSourceFiles рекурсивно знаходить у вказаному каталозі всі вихідні файли, ігноруючи директорії та файли, що відповідають відомим шаблонам (включаючи .git та node_modules).

## Публічний API

measureCoveragePerFile — Обчислює покриття коду для кожного файлу, використовуючи Vitest із генератором JSON, і одночасно виявляє будь-які провалені тести.
getUncoveredFiles — Визначає файли, які не досягли заданого рівня покриття.
findSourceFiles — Рекурсивно шукає файли з вихідним кодом у заданій директорії, ігноруючи тестові файли та проігноровані каталоги, для ініціалізації, якщо дані про покриття відсутні.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- Кешує результати в межах одного прогону.
- Свідомо пропускає шляхи: `.git`, `node_modules`.
