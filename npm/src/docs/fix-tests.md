## Огляд

Цей файл виявляє падаючі юніт-тести, використовуючи `vitest` у режимі виводу JSON, та описує їх помилки, спираючись на `package.json` та `results.json`. Він автоматично виправляє ці тести шляхом взаємодії з LLM через pi SDK у текстовому режимі (`callText`), що усуває проблеми з точним збігом рядків при порівнянні URL, зворотних апострофів та спецсимволів.

## Поведінка

Поведінка:
getFailingTests виявляє падаючі unit-тести, запускаючи vitest у JSON-режимі та повертаючи список файлів разом із повідомленнями про помилки.
buildFixTestsPrompt створює повний промпт для агента, який містить контекст падаючих тестів, їхні помилки, вміст тест-файлів та код із джерела для допомоги LLM.
fixFailingTests ітерує падаючі тести, звертається до LLM через pi SDK для виправлення та безпосередньо записує змінений код у файли, повторюючи спробу до максимуму, поки тести не почнуть проходити.

## Публічний API

getFailingTests — Запускає юніт-тести (vitest) у форматі JSON і повертає перелік файлів, що не пройшли тестування, разом з відповідними повідомленнями про помилки.
buildFixTestsPrompt — Створює інструкцію (промпт) для виправлення одного файлу з тестами, надаючи вміст цього файлу безпосередньо у промпт.
fixFailingTests — Виявляє та виправляє неякісно пройдені тести, використовуючи текстовий режим pi SDK та пряме записування файлу. Негайно повертає, якщо всі тести вже успішно проходять.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
