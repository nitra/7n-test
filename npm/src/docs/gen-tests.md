## Огляд

Файл реалізує генерацію тестових файлів для експортів проєкту, використовуючи стратегію багатоетапної маршрутизації на основі складності кожного експорту. Експорти класифікуються на тривіальні/прості та складні. Для простих випадків використовується локальна модель pi, а для складних — хмарна. Здійснюється генерація спільного заголовка (імпорти, моки, налаштування) за допомогою хмарної моделі. Потім генеруються блоки опису для кожного експорту з урахуванням його складності. Всі комунікації (локальні та хмарні) здійснюються через pi SDK; відсутність конфігурації локальної моделі або відсутність експортів призводить до генерації у режимі однофайлового хмарного запиту. При помилках функція не генерує винятків, а повертає порожнє значення, забезпечуючи механізм fail-safe.

## Поведінка

findTestRules шукає правила тестів проєкту, обходячи батьківські каталоги до чотирьох рівнів.
startRe — це регулярний вираз для ідентифікації викликів функцій на верхньому рівні модуля.
buildGenTestsPrompt створює текстовий опис для генерації тестів на основі наданих файлів, відображаючи їхнє покриття та зміст.
generateTests генерує тестові файли для наданих файлів, використовуючи багатоетапну маршрутизацію: об'єднання спільного заголовка за допомогою хмарної моделі та генерацію блоків `describe` з використанням локальної моделі з поверненням до хмари у разі проблем.

## Публічний API

findTestRules — Знаходить файли правил `n-test.mdc` для проекту, піднімаючись від поточної директорії на максимальні 4 рівні.
startRe — Запускає процес відтворення або реінсталяції (з наміру файлу).
buildGenTestsPrompt — Формує узагальнений текстовий опис для генерації тестів, який відображається лише під час тестування.
generateTests — Створює тестові сценарії для всіх вказаних файлів, використовуючи багатоступеневе направлення виводу через локальну LLM, якщо вона доступна, або переключаючись на генерацію для одного файлу у хмарі в іншому випадку.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
