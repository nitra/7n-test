## Огляд

Цей модуль генерує тестові файли, застосовуючи стратегію "Test generation via pi SDK with per-export tiered routing". Стратегія передбачає класифікацію кожного експорту (trivial/simple або complex). Для простих експортів спочатку використовується локальна модель (яка обирається через `opts.localModel` або `N_TEST_LOCAL_MODEL` env var), а для складних або при відсутності локальної моделі — хмарне API. Процес включає генерацію спільного заголовка (imports, mocks, setup) через хмару, а також генерацію блоків `describe` для кожного експорту з маршрутизацією за складністю. Згенеровані локально блоки підлягають валідації; у разі виявлення антипатернів відбувається відкат до генерації хмарою. Усі комунікації (локальні та хмарні) здійснюються виключно через pi SDK. Функції такі, як `findTestRules`, `startRe`, `buildGenTestsPrompt` та `generateTests` керують цим процесом, причому система має механізм fail-safe, який не виводить винятків та повертає порожнє значення при певних помилках.

## Поведінка

Поведінка
findTestRules знаходить конфігураційні правила для тестування, шукаючи файл `.cursor/rules/n-test.mdc` у каталогах вище поточної директорії до чотирьох рівнів.
startRe — це регулярний вираз, який ідентифікує виклики функцій на рівні модуля, що виконуються при завантаженні.
buildGenTestsPrompt генерує текстовий звіт для тестування всіх файлів, включаючи їхній покриття та вміст.
generateTests генерує тестові файли для вказаних джерел, використовуючи багаторівневий підхід: спробує застосувати локальну модель для блоків простих експортів з відмовостійкістю до хмарного API, або виконає генерацію в режимі одного файлу за допомогою хмарного API, якщо локальна модель недоступна або немає експортів.

## Публічний API

- findTestRules — Знаходить правила для юніт-тестів, піднімаючись у дереві каталогів до чотирьох рівнів.
- startRe — Починає перевірку повторності або повторного запуску.
- buildGenTestsPrompt — Формує звітний підсумок-промпт, який відображається в тестах.
- generateTests — Створює тестові сценарії для вказаних файлів, використовуючи багатоетапну маршрутизацію, якщо локальна LLM доступна, і відмовляється до однофайлової генерації в хмарі.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
- За певних помилок повертає порожнє значення (напр. `null`) замість винятку.
