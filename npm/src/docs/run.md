## Огляд

Main auto-test loop забезпечує автоматизований цикл тестування з метою підвищення покриття коду. У першій фазі відбувається вимірювання покриття коду по файлах через vitest+lcov. Якщо покриття нижче заданого порогу, генеруються тести для відповідних файлів. Процес повторюється, доки покриття не досягне максимуму або не буде досягнуто обмежень ітерацій. Друга фаза включає мутаційне тестування, яке проводиться з використанням існуючих провайдерів покриття, залежність від .n-cursor.json. Після виявлення виживших мутантів, pi agent автоматично їх виправляє. Функція runAutoTest перехоплює можливі помилки, запобігаючи виняткам на виході, але не виконує кешування.

## Поведінка

Поведінка
1. Викликається `runAutoTest` для запуску циклу автоматичного тестування.
2. У циклі до п'яти ітерацій вимірюється покриття кожної функції у проекті.
3. Якщо існують падіння тестів, вони автоматично виправляються через агент, і цикл продовжується.
4. Якщо немає тестів, відбувається запуск процесу Bootstrap: скануються вихідні файли, класифікуються, і для тих, що потребують тестів, генеруються початкові тести.
5. Обчислюється загальне покриття. Якщо воно досягло встановленого порогу, цикл зупиняється.
6. Якщо покриття не покращилося порівняно з попередньою ітерацією, цикл зупиняється.
7. Якщо покриття недостатнє, генеруються тести для некритичних файлів і цикл повторюється.
8. Після завершення циклу, якщо не вказано інакше, запускається мутаційне тестування, яке залежить від конфігурації в `.n-cursor.json`.
9. У разі успішного виживання мутантів, виконується повторний запуск покриття для перевірки.

## Гарантії поведінки

- Перехоплює помилки і не пропускає винятків назовні (fail-safe).
