## Огляд

Робота цього файлу полягає у генерації унікального цифрового відбитка поточного стану робочого дерева проєкту за допомогою функції `worktreeFingerprint`. Він збирає інформацію про стан коміту Git та контент файлів, що ще не додані до відстеження. Це дозволяє однозначно ідентифікувати консистенцію робочої копії репозиторію.

## Поведінка

1. Обчислення консистенції робочого дерева.
2. Зчитування хешу поточного коміту Git.
3. Зчитування тексту відмінностей між комітом і робочим деревом.
4. Зчитування списку невідстежуваних файлів Git.
5. Для кожного невідстежуваного файлу обчислюється його хеш.
6. Утворюється відсортований список пар, що містять назву файлу та його хеш.
7. Усі зібрані дані (хеш коміту, текст відмінностей, відсортовані пари невідстежуваних файлів) конкатенуються в один текстовий блок.
8. Від цього текстового блоку обчислюється SHA256 хеш.
9. Якщо будь-яка частина процесу не може бути виконана (наприклад, поточний каталог не є Git-репозиторієм), функція повертає null.

## Публічний API

worktreeFingerprint — визначає унікальний відбиток поточного стану git-робочого дерева.
